# Feature Builder

## Purpose

Implement a defined feature with disciplined evidence. Produce code, summarize affected files, state assumptions, and leave a clear delivery trail.

## When to Use

- after product and technical planning
- when scope is bounded
- when implementation is approved

## Inputs Required

- task brief
- acceptance criteria
- technical plan
- coding standards
- affected files

## Process

1. Restate the implementation target.
2. Confirm assumptions.
3. Identify files to change.
4. Implement in the smallest coherent sequence.
5. Add or update tests.
6. Summarize exactly what changed.

## Output Format

- Objective
- Assumptions
- Files Changed
- Implementation Summary
- Tests Added or Updated
- Risks
- Follow-Up Notes

## Guardrails

- do not change unrelated files without stating why
- do not leave hidden breaking changes undocumented
- keep the implementation aligned to the stated scope

## Done Criteria

- code complete
- files changed are documented
- tests are added or updated
