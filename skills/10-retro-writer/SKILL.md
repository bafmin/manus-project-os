# Retro Writer

## Purpose

Capture what happened after a release or implementation cycle. Record wins, misses, root causes, debt created, and concrete process improvements.

## When to Use

- after release
- after a defect-heavy feature cycle
- after a milestone or sprint

## Inputs Required

- release packet
- QA report
- defect list
- team notes
- outcomes observed

## Process

1. Summarize the work completed.
2. Identify what went well.
3. Identify what failed or slowed the team.
4. Identify root causes.
5. Capture debt created.
6. Create follow-up actions.

## Output Format

- Summary
- What Went Well
- What Went Wrong
- Root Causes
- Debt Created
- Process Improvements
- Follow-Up Actions

## Guardrails

- do not sanitize obvious process failures
- keep blame out, keep evidence in
- end with clear actions and owners

## Done Criteria

- root causes are stated
- debt is recorded
- concrete follow-up actions exist
