# Definition of Done

A task is done only when all conditions below are true.

## Delivery

- The requested behavior works.
- Acceptance criteria are met.
- Edge cases were considered.

## Code Quality

- Code follows project standards.
- Related code paths were reviewed.
- No known critical shortcuts remain undocumented.

## Testing

- Required tests were added or updated.
- Relevant test suites pass.
- Manual or browser verification is complete where needed.

## Documentation

- User-facing behavior changes are documented.
- Technical docs are updated if architecture changed.
- Release notes draft is prepared.

## Risk and Release

- Risks are recorded.
- Rollback approach exists.
- Migration steps are documented if needed.
