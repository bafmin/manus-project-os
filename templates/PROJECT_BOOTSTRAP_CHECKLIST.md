# Project Bootstrap Checklist

This checklist prepares a new project to operate under the Manus Project OS.

A project should not begin development until this checklist is completed.

---

## Foundation

Required governance documents must exist.

- project charter completed
- product requirements defined
- technical architecture documented
- engineering standards defined
- security rules defined
- test policy defined
- release process defined

Artifacts should live in:

/docs/core  
/docs/architecture

---

## Skills Setup

Skills must be configured for the project stack.

- base skills copied into project
- skill language adapted to project stack
- project examples added
- output contracts verified
- guardrails reviewed

Skills should exist in:

/skills

---

## Delivery Controls

Operational controls must be established.

- definition of done approved
- decision log started
- risk register started
- QA checklist defined
- release notes template reviewed
- rollback strategy defined

Artifacts stored in:

/docs/decisions  
/docs/risks

---

## Engineering Setup

Engineering infrastructure must be ready.

- repository initialized
- branching strategy defined
- environments defined
- CI checks configured
- automated tests enabled
- secret management defined
- staging environment available

---

## Observability Setup

Systems must support debugging and monitoring.

- application logging defined
- error tracking configured
- monitoring strategy defined
- performance metrics defined

---

## Readiness Check

Before development begins confirm:

- first feature request documented
- first change plan created
- first task brief created
- QA workflow defined
- release path documented

When all items are complete the project is ready for development.
